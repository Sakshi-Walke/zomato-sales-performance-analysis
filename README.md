# zomato-sales-performance & User Analytics Dashboard


##  Project Overview
This project presents an end-to-end data analysis of Zomato's sales, user behavior, and city-level performance using Power BI.

The dashboard helps stakeholders understand revenue trends, user engagement, and operational performance across 800+ cities.

---

##  Business Problem
Zomato operates across multiple cities but lacks a centralized system to:
- Track revenue performance
- Monitor user growth and churn
- Analyze city-level contributions
- Evaluate food category performance

 This project converts raw data into actionable business insights.

---

##  Dataset Summary
-  Total Revenue: ₹987M  
-  Total Orders: 301K  
-  Total Users: 78K  
-  Ratings: 148K  
-  Cities Covered: 822  

---

##  Dashboard Features

### Sales Overview
- Year-wise sales trend (2017–2020)
- Peak sales in 2019 (~₹0.42bn)
- Drop observed in 2020

###  City Performance
Top Cities by Revenue:
- Tirupati – ₹43M  
- Electronic City (Bangalore) – ₹29M  
- Baner (Pune) – ₹27M  
- Raipur – ₹23M  

 Insight: Tier-2 cities contribute significantly to revenue

---

###  User Analysis
- Total Users: 78K
- Active Users: 78K
- User Gain vs Loss:
  - Gained: ~33K
  - Lost: ~33K+

 Insight: High churn rate across cities

---

###  Food Category Analysis
- Non-Veg: ₹122M  
- Veg: ₹106M  
- Other: ₹24M  

👉 Insight: Non-veg dominates sales

---

###  Ratings & Engagement
- Ratings strongly correlate with active users
- Cities with higher ratings show better engagement

---

##  Key Insights
-  Revenue declined after 2019
-  Few cities drive majority of revenue
-  High user churn observed
- Non-veg category performs best
- Ratings influence engagement

---

##  Business Recommendations
- Improve user retention strategies
- Focus marketing on top-performing cities
- Improve low-rated restaurants
- Promote underperforming categories
